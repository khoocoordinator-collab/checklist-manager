# SCOPE.md — Checklist Manager

## Core Concept
Offline-first web app for outlet operations. Field staff fill checklists on mobile and sync manually when online. Supervisors verify completed checklists. Admins manage templates, schedules, and outlets on desktop.

## User Flows

**Field Staff (Mobile):**
1. Enter team passcode → see pending checklists
2. Tap checklist → fill items (yes/no, number, text, photo) → sign off
3. Save locally → tap "Sync" when online

**Supervisor (Mobile):**
1. Enter supervisor passcode → see completed checklists awaiting verification
2. Review items, photos, and staff signature
3. Enter name → sign off → checklist marked as verified

**Admin (Desktop):**
1. Login → manage outlets, teams, templates, schedules
2. Generate instances manually or let cron auto-generate
3. View completion and verification history

## Data Model

- **Outlet:** name, location — groups teams by location
- **Team:** name, passcode, type (staff/supervisor), outlet
- **Template:** title, items[], schedule, team, validity windows, requires_supervisor flag
- **TemplateItem:** text (max 48 chars), order, response_type (yes_no/number/text/photo), is_required
- **Schedule:** frequency (daily/weekly/bi-weekly/monthly), time_of_day, day_of_week, day_of_month — auto-creates pending instances 1 hour before scheduled time
- **Instance:** template snapshot, team, date_label, status (draft/pending/completed/verified/expired), supervisor fields
- **InstanceItem:** text snapshot, response_type, response_value, is_checked, notes, photo
- **Signature:** base64 PNG, signed_by, signed_at — one per instance (staff)

## Key Behaviours

| Scenario | Behaviour |
|----------|-----------|
| Offline | Full fill functionality via localStorage |
| Sync conflict | Last write wins |
| Template deleted/hidden | Existing instances preserved (snapshot data) |
| Schedule fires | Cron creates "pending" instance 1 hour before scheduled time |
| Checklist expires | Status written to DB if not completed within validity window |
| Supervisor required | Instance assigned a supervisor team; supervisor must sign off |
| Supervisor scope | Supervisors only see checklists from their own outlet |
| Data retention | Instances auto-deleted after 90 days |
